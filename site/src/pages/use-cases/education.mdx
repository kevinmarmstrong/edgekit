A study assistant that knows the course material and can answer student questions at any hour is transformative. But cloud-based AI tutors create problems: per-query costs scale with class size, student data flows to third-party servers, and the system stops working without an internet connection.

Edgekit builds study assistants that run entirely on the student's device. The course material is indexed at deploy time, the model runs on the student's GPU, and every interaction is private by construction.

## The problem

Educational institutions face a tension between AI adoption and student privacy:

**Cost scaling.** A class of 200 students asking 10 questions each per week at $0.003 per query adds up fast. Multiply by every class in a department. Most schools cannot budget for per-query AI costs.

**Privacy regulations.** FERPA, COPPA, GDPR, and institutional policies restrict how student data can be processed. Cloud AI services require data processing agreements, privacy impact assessments, and ongoing compliance monitoring.

**Equity.** Cloud AI requires reliable internet access. Students studying at home, on public transit, or in areas with poor connectivity are excluded from the tools their peers use on campus.

**Control.** Instructors cannot control what a general-purpose AI says about their subject. Cloud models may give answers that contradict the course material or introduce concepts the instructor has not covered yet.

## The solution

Edgekit lets instructors create study assistants scoped to their course material. The AI only answers from the indexed content -- lecture notes, textbook excerpts, assignment descriptions, and supplementary readings.

Example interactions:

- "Explain the difference between stack and heap allocation from the lecture notes" -- Answers from the indexed lecture content
- "What are the key themes from this week's readings?" -- Synthesizes across multiple indexed documents
- "Help me understand the formula on slide 34" -- References specific course material
- "What should I focus on for the midterm based on what we covered?" -- Draws from all indexed materials

Every answer cites the source material, so students can verify claims against the original documents.

## Implementation

### Step 1: Index course materials

Organize course content as markdown files:

```
content/
  lectures/
    week-01-introduction.md
    week-02-data-structures.md
    week-03-algorithms.md
  readings/
    chapter-01-summary.md
    chapter-02-summary.md
  assignments/
    homework-01.md
    homework-02.md
```

Build the index:

```bash
npx @edgekit/cli build \
  --input ./content \
  --output ./public/content-index.json
```

### Step 2: Embed the assistant

```html
<div id="study-assistant"></div>

<script type="module">
  import { createRuntime } from '@edgekit/core'
  import { webllm } from '@edgekit/model-webllm'
  import { localRAG } from '@edgekit/rag-local'
  import { webComponent } from '@edgekit/ui-component'

  const ui = webComponent({
    theme: 'auto',
    placeholder: 'Ask about the course material...',
  })

  const runtime = createRuntime({
    model: webllm({ tier: 'standard' }),
    rag: localRAG({ indexUrl: '/content-index.json', topK: 5 }),
    ui,
    downloadPolicy: 'prompt',
    systemPrompt: `You are a study assistant for this course.
      Answer questions using only the provided course materials.
      Cite the lecture or reading where you found the information.
      If the materials do not cover a topic, say "This is not
      covered in the current course materials." Do not speculate
      or provide information from outside the course content.`,
    guardrails: {
      maxInputTokens: 300,
      maxOutputTokens: 800,
      maxConversationTurns: 30,
    },
  })

  ui.mount(document.getElementById('study-assistant'), runtime)
</script>
```

### Step 3: Deploy as a static page

Host the study assistant on your course website or LMS. The content index is a static JSON file served alongside the page. No server infrastructure is needed.

```json
{
  "scripts": {
    "build": "npx @edgekit/cli build && cp -r public/ deploy/"
  }
}
```

Update the index when course materials change. Students' browsers cache the model weights, so they only download the new content index (typically under 5 MB).

## What students experience

**First visit:** The student sees a chat interface on the course page. When they ask a question, they are prompted to download the AI model. After a one-time download (~2.5 GB), the assistant is ready.

**Studying:** The assistant answers questions about course material in 1-3 seconds. Students can ask follow-up questions, request clarifications, or ask the assistant to explain concepts in different ways. Every answer references the specific lecture or reading.

**Offline:** After the initial model download, the assistant works without internet. Students can study on the bus, in the library, or anywhere else.

## Privacy and compliance

Edgekit's architecture makes compliance straightforward:

| Concern | Cloud AI | Edgekit |
|---|---|---|
| Student query data | Sent to third-party servers | Never leaves the device |
| FERPA compliance | Requires data processing agreement | No student data is processed externally |
| COPPA (under 13) | Complex consent requirements | No data collection to consent to |
| Data retention | Vendor-dependent policies | Entirely student-controlled |
| Breach risk | Vendor's security posture | No central data store to breach |

There is no student data to protect because there is no student data to collect. The strongest privacy guarantee is architectural: the data never leaves the device in the first place.

## Use cases within education

**Lecture review assistants.** Index lecture transcripts and slides. Students review material by asking questions instead of re-reading notes.

**Textbook companions.** Index textbook chapters (with appropriate licensing). Students get an AI tutor scoped to the textbook content.

**Lab manuals.** Index lab procedures, safety protocols, and equipment guides. Students get instant answers to procedural questions during lab sessions.

**Language learning.** Index vocabulary lists, grammar rules, and example sentences. Students practice by having conversations grounded in the learning material.

## Instructor benefits

**Consistent answers.** The AI only answers from indexed materials, so it cannot contradict the instructor or introduce concepts out of sequence.

**Zero operational cost.** No API keys to manage, no usage bills to monitor, no per-student cost calculations.

**Update control.** The instructor controls exactly what is in the index. Adding or removing content is a rebuild and redeploy.

**Analytics-free by default.** No student interaction data is collected unless the instructor explicitly adds client-side analytics.
